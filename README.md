# Candidate Search System

## Documentation

- **Decision log**: `docs/DECISIONS.md`
- **Evaluation notes**: `docs/EVAL.md`
- **Batch evaluation output**: `analysis_results.md` (generated by `batch_eval.py`)

## Setup

### 1) Create a virtual environment

```powershell
py -m venv .venv
.\.venv\Scripts\python -m pip install -U pip
.\.venv\Scripts\python -m pip install pandas openpyxl sentence-transformers faiss-cpu google-generativeai rich
```

### 2) Configure Gemini

Set an environment variable (recommended):

```powershell
$env:GEMINI_API_KEY="YOUR_KEY_HERE"
```

Alternatively, create a `.env` file in the project root:

```bash
GEMINI_API_KEY=YOUR_KEY_HERE
```

## Run the CLI

```powershell
.\.venv\Scripts\python main.py "Senior Python dev in Bangalore, 4+ years, not a job hopper"
```

Optional flags:

- `--excel` to point to a different Excel file
- `--index-dir` to control where FAISS artifacts are stored
- `--rebuild` to force rebuilding embeddings

Example:

```powershell
.\.venv\Scripts\python main.py --excel "data\Candidates and Jobs.xlsx" --rebuild "Senior Python dev in Bangalore, 4+ years"
```

## Run batch evaluation

This generates `analysis_results.md` with Top 5 results per query:

```powershell
.\.venv\Scripts\python batch_eval.py
```

You can also override paths:

```powershell
.\.venv\Scripts\python batch_eval.py --excel "data\Candidates and Jobs.xlsx" --index-dir "artifacts\candidate_index"
```
